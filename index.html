Propuesta 1: Sistema de Control de Asistencia

Implementación de un sistema digital de control de asistencias basado en códigos QR.

Justificación:

Actualmente, las asistencias se registran manualmente en papel, lo cual es ineficiente para más de 500 estudiantes. Un sistema digital permitirá un registro rápido y fiable.

Objetivos:

  • Implementar un sistema digital de control de asistencias.
  • Mejorar la transparencia y precisión en el seguimiento de la asistencia.

Metodología:

  • Desarrollar una versión mejorada del sistema existente (versión 2.0).
  • Iniciar la implementación durante el primer mes del año académico.

Propuesta 2: Sistema de Matrícula Digital para Laboratorios de la EPIS

Desarrollo e implementación de un sistema de matrícula digital para los laboratorios.

Justificación:

El proceso actual de matrícula en hojas de Excel genera largas colas y es ineficiente. Un sistema digital optimizará la gestión de los laboratorios.

Objetivo:

Desarrollar e implementar un sistema de matrícula digital que permita a los estudiantes registrarse de forma rápida y sencilla para los laboratorios.

Metodología:

  • Implementar un sistema basado en un prototipo ya desarrollado en Figma.
  • Finalizar la implementación en los primeros tres o cuatro meses del semestre académico.

Propuesta 3: Sistema de Préstamo de Libros para la Biblioteca del CEIS

Implementación de un sistema digital para el registro y control del préstamo de libros.

Justificación:

El préstamo de libros se gestiona actualmente en hojas de papel, lo cual es poco práctico y eficiente. Un sistema digital mejorará la eficiencia y facilitará el acceso a los recursos académicos.

Objetivo:

Implementar un sistema digital para el registro y control del préstamo de libros en la biblioteca del CEIS.

Metodología:

  • Desarrollar e implementar un sistema que digitalice el proceso de préstamo de libros.
  • Implementación durante el primer mes del año académico.
  • Uso de tecnología QR y tarjetas de identificación para registrar el préstamo y la devolución de libros.

Propuesta 4: Correo Institucional para el CEIS

Implementación de un correo institucional para centralizar las comunicaciones y solicitudes.

Justificación:

Las comunicaciones actuales a través de WhatsApp dificultan el seguimiento y la formalización de las solicitudes. Un correo institucional garantizará una comunicación fluida y profesional.

Objetivo:

Implementar un correo institucional (con dominio ceis@unsch.edu.pe) para centralizar las comunicaciones y solicitudes de los estudiantes hacia el CEIS.

Metodología:

Realizar los trámites correspondientes para obtener el correo institucional durante la primera semana del período académico.

Propuesta 5: Sistema de Control Financiero para el CEIS

Implementación de un sistema para gestionar la contabilidad del CEIS de manera digital.

Justificación:

La falta de un sistema eficiente para gestionar la contabilidad complica el control del presupuesto, los ingresos y los gastos. La digitalización garantizará una gestión más precisa y transparente.

Objetivos:

  • Implementar un sistema de control financiero para registrar y gestionar los ingresos y egresos del CEIS de manera digital.
  • Facilitar la creación de informes financieros para la toma de decisiones estratégicas.

Metodología:

  • Desarrollar un software para registrar las transacciones y generar reportes financieros automáticos.
  • Realizar capacitaciones para la directiva y el tesorero sobre el uso del sistema.
  • Implementar el sistema durante el segundo trimestre del año académico.

Propuesta 6: Sistema de Inventario para el CEIS

Desarrollo de un sistema digital para gestionar el inventario del CEIS.

Justificación:

El inventario actual en hojas de papel facilita la pérdida de información y dificulta el control de responsabilidades. Un sistema digital permitirá un mejor control y mantenimiento de los recursos.

Objetivo:

Desarrollar un sistema digital para gestionar el inventario del CEIS, permitiendo un control más eficiente de los recursos y sus responsables.

Metodología:

  • Realizar un análisis de requerimientos y desarrollar el sistema durante cinco o seis meses.
  • Si no es posible completar el sistema, dejar el proyecto en fase avanzada para la siguiente gestión.

Propuesta 7: Elaboración de Poleras Americanas (estilo bomber) para la Carrera de Ingeniería de Sistemas

Diseño y confección de poleras distintivas para los estudiantes de Ingeniería de Sistemas.

Justificación:

La carrera carece de un símbolo distintivo que refuerce el sentido de pertenencia. Las poleras tipo "Bomber" brindarán identidad y servirán como vestimenta adecuada para eventos académicos y deportivos.

Objetivo:

Diseñar y confeccionar poleras para los estudiantes de Ingeniería de Sistemas mediante un concurso de diseño participativo.

Metodología:

  1. Realizar un concurso de diseño abierto a todos los estudiantes durante las dos primeras semanas del año académico.
  2. Votación virtual para elegir el diseño ganador durante la tercera semana.
  3. Cotización y confección de las poleras para los estudiantes interesados.
